# HELIOS GUARD - Technical Documentation

## Project Description

**Helios Guard** - это MVP-система защиты спутников от солнечных бурь и космической погоды. Система реализует полный цикл обнаружения, анализа и защитого маневра.

### Назначение

Система предназначена для демонстрации концепции TinyML-based защита спутников:
- Мониторинг солнечной активности в реальном времени
- Раннее обнаружение опасных событий (вспышки Х-Класса)
- Расчет времени прибытия частиц (ТоА)
- Автоматическое выполнение защитных команд

### Компоненты

1. **SensorUnit** (`SensorUnit` class)
- Генерирует симулированные данные солнечной активности
- Классы вспышек: A, B, C, M, X
- Скорость СМE: 300-3000 km/s

2. **TinyMl_Trigger** (`TinyML_Trigger` class)
- Пороговый детектор (Threshold potential: 1e-4 W/m²)
- Режимы: Sleep (0.01w) / Active (2.0w)
- Активирует AI при превышении порога

3. **MissionAI** (`MissionAI` class)
- Анализ угрозы
- Расчет времени прибытия: T = D / V
- Генерация защитных команд

## Physics

### Расчет времени прибытия (ТоА)

```
T = ( D / V ) - ( D / c )

Где:
    D - расстояние до солнца = 150,000,000 км
    V - скорость солнечных частиц (км/с)
    T - время прибытия (с)
    c - Скорость света в вакууме = 300,000 км/с 
```

### Примеры

| Скорость СМЕ | Время до прибытия |
|--------------|-------------------|
| 500 km/s | 83.3 часа (3.5 суток) |
| 1000 km/s | 41.7 часа (1.7 суток) |
| 2000 km/s | 20.8 часа |
| 3000 km/s | 13.9 часа |

### Классификация вспышек

| Класс | Поток (W/m²) | Опастность |
|-------|--------------|------------|
| A | < 1e-7 | Минимальная |
| B | 1e-7 - 1e-6 | Низкая |
| C | 1e-6 - 1e-5 | Умеренная |
| M | 1e-5 - 1e-4 | Высокая |
| X | > 1e-4 | Критическая |

## Расширение системы

### Добавление новых компонентов

1. Добавьте новый класс в `main.py`
2. Инициализируйте в `HeliosGuardSystem.__init__`
3. Интегрируйте в цикл `run_detection_cycle`

### Интеграция с реальными данными

Замените `SensorUnit.generate_reading()` на:
- NASA DONKI API
- NOAA SWPC API
- GOES satellite data

### Переход на реальное TinyML

Замените пороговый триггер на:
- TensorFlow Lite Micro
- PyTorch Mobile
- ONNX Runtime

## Логирование

Система использует стандартный Python `logging` с timestamp:

```
19:25:30.055 | WARNING | >>> THRESHOLD EXCEEDED! X-ray flux: 6.94e-04 W/m²
19:25:30.055 | INFO | >>> TinyML_Trigger: Waking AI system (power: 0.01W -> 2.0W)
19:25:30.055 | WARNING | !!! COMMAND #1: PROTECTIVE_MANEUVER_CRITICAL
```

### Уровни логирования

| Уровень | Использование |
|---------|---------------|
| DEBUG | Детальная отладка |
| INFO | Нормальная работа |
| WARNING | Предупреждения, порог превышен |
| ERROR | Ошибки системы |
